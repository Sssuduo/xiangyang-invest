# V15.3 活动台账录音总结全功能修复 + 服务器清理（2026-07-15）

## 一、Bug 修复（均已上线生产）

### 1. 文本校正页跳转失败
- **现象**：点"文本校正"按钮，页面重定向回活动台账
- **根因**：路由器是 `createWebHistory()`（history 模式），但按钮造的是 `#/admin/...` hash URL，history 模式不识别 `#`
- **修复**：`openTextCorrection` 改用 `router.resolve({ name: 'TextCorrection', params: { id } })` 生成真实路径
- **commit**: `1e146fe`

### 2. 录音文件无法上传（首文件无入口）
- **现象**：新建台账时无法上传第一个录音文件
- **根因**：唯一的 `el-upload` 被 `v-if="audioFiles.length > 0"` 卡住，首文件时整个操作行隐藏
- **修复**：合并原双 el-upload 为单组件，上传入口独立于文件有无渲染（仅处理中时隐藏）；统一用 `audioFile` 做"有文件"判断
- **commit**: `1e146fe`

### 3. 下载 Word 404
- **现象**：查看抽屉中"下载 Word"按钮 404
- **根因**：保存端落到 `backend/static/meetings/`，读取端（API + 查看抽屉）从 `<项目根>/static/meetings/` 读，两端分裂
- **修复**：保存端 `_ensure_dir()` + 读取端 `download_audio_docx()` 统一对齐到 `current_app.static_folder`（Flask 单一真来源）；旧文件迁移到新路径
- **commit**: `5326235`

### 4. 文本校正页打开无数据
- **现象**：业务用户打开 `/admin/text-correction/:id` 后页面空白
- **根因**：路由设了 `meta: { requiresAuth: true }`（仅 admin），业务用户被路由守卫重定向到 `/admin/login`；但该页后端 API 用 `@dual_login_required`（允许 admin 或 business），是前端守卫比后端更严
- **修复**：新增 `requiresDualAuth` 路由元数据，守卫同时校验 admin/business，任一通过即放行；TextCorrection 路由改用 `requiresDualAuth`
- **commit**: `95cdf0e`

### 5. 文本校正页 500 错误
- **现象**：打开校正页报 500
- **根因**：`textCorrection.js` API 路径写 `/api/admin/...`，与 `baseURL: '/api'` 拼成 `/api/api/admin/...`；其他 API 文件（activityLedger 等）路径都正确用 `/admin/...`，独独 textCorrection.js 带双重前缀
- **修复**：去掉路径里的 `/api` 前缀
- **commit**: `3feb7c9`

### 6. 保存后文本不替换（核心功能缺失）
- **现象**：选中词 → 输入替换词 → 点确认 → 保存成功，但刷新后仍是原词
- **根因链条**：
  1. `confirmReplace` 只关闭对话框，**从未把 correction 加入 `corrections` 数组**（第一版修复）
  2. `saveCorrectionsApi` 用 `import('axios')` 裸实例（无 baseURL/无 cookie/无拦截器）→ 后端 405（第二版修复）
  3. 生产 `admin_voice_knowledge.py` 的 `save_text_corrections` **只写 TextCorrectionRecord 记录，不替换台账文本字段**（第三版修复）
- **修复**：
  - 前端 `confirmReplace`：把 correction unshift 进数组 + 同步替换编辑器 DOM
  - 前端 `saveCorrectionsApi`：改用已 import 的 `saveCorrections`（共享 api 实例）
  - 后端 `save_text_corrections`：保存后遍历 corrections，对 4 个文本字段做 `original→replacement` 字符串替换 + `db.session.commit()`
- **commits**: `f96781e`, `daa013a`, `fca7d65`

### 7. 选词长度限制
- **现象**：2 字词语无法选中替换
- **根因**：`TextEditor.handleMouseUp` 里 `text.length < 3` 直接 return（不 emit selection-change）
- **修复**：`< 3` → `< 2`
- **commit**: `eb8adeb`

### 8. 编辑器文本框无法滚动
- **现象**：长文本看不到全部内容，无滚动条
- **根因**：`.text-editor` 的 `max-height` 被去掉后内容撑开容器，无滚动约束
- **修复**：恢复 `max-height: calc(100vh - 320px)` + `overflow-y: auto`
- **commit**: `627e825`

### 9. 编辑器替换后不更新
- **现象**：点替换按钮后编辑器内文本不变
- **根因**：`doReplace` 里 `window.getSelection()` 失效（focus 移到 popover input 后 selection 丢失）
- **修复**：`handleMouseUp` 时保存 range 引用到 `savedRange`，`doReplace` 用它重选举区并替换
- **commit**: `59d85cf`

## 二、服务器磁盘清理

### 清理前状态
- 磁盘：22G / 40G（59%），曾两次撑爆（92%/100%）导致 git fetch 失败

### deploy.sh 递归备份 bug 修复
- **根因**：备份输出到 `APP_DIR/backups/`，但 tar 源目录是 `APP_DIR`（含 backups），每次备份包含上一次 → 指数膨胀（4.3G→8.5G→…）
- **修复**：tar 加 `--exclude='static/uploads'` `--exclude='static/meetings'` `--exclude='backend/static'` `--exclude='backups'`
- **效果**：代码备份从 8.5G → 1.5M

### FunASR/PyTorch 生态清理
- **发现**：funasr pip 包已卸载，但遗留的 PyTorch/CUDA 依赖（5.9G）仍在
- **清理**：
  - nvidia CUDA（cu13/cudnn/nccl/cusparselt/nvshmem）：2.7G
  - torch + triton + functorch：1.9G
- **安全依据**：生产代码不 import torch/nvidia/triton；gunicorn 内存未加载；requirements.txt 无依赖

### 清理后状态
- 磁盘：18G / 40G（46%），可用 21G
- invest-app + nginx 服务正常运行
